---
title: "Safety checker"
description: "Validate waypoints, takeoff altitude, and speed against geofences before commanding the vehicle, using SafetyCheckerServer and SafetyCheckerClient."
---

The safety checker system validates vehicle commands against geofence rules before they are issued. It runs as a separate process (`SafetyCheckerServer`) that your script talks to through a client (`SafetyCheckerClient`).

## Architecture

| Component | Role |
|-----------|------|
| `SafetyCheckerServer` | Loads geofence YAML config and KML files; validates requests over ZMQ REQ/REP |
| `SafetyCheckerClient` | Sends validation requests from your script to the server |
| `NoOpSafetyChecker` (v2) | Passthrough implementation used when no server is configured |

    ## v1 setup

    ### 1. Create a geofence config

    ```yaml
    # geofence_config.yaml
    vehicle_type: copter
    max_speed: 15
    min_speed: 0.5
    max_alt: 120
    min_alt: 5
    include_geofences:
      - flight_area.kml
    exclude_geofences:
      - no_fly_zone.kml
    ```

    | Parameter | Type | Description |
    |-----------|------|-------------|
    | `vehicle_type` | string | `copter` or `rover` |
    | `max_speed` | float | Maximum allowed speed (m/s) |
    | `min_speed` | float | Minimum allowed speed (m/s) |
    | `max_alt` | float | Maximum altitude for copters (m) |
    | `min_alt` | float | Minimum altitude for copters (m) |
    | `include_geofences` | list | KML files defining allowed areas |
    | `exclude_geofences` | list | KML files defining no-go zones |

    ### 2. Start the server

    ```bash
    python -m aerpawlib.v1.safety --port 14580 --vehicle_config geofence_config.yaml
    ```

    Or start it from Python (the constructor blocks until terminated):

    ```python
    from aerpawlib.v1.safety import SafetyCheckerServer

    server = SafetyCheckerServer("geofence_config.yaml", server_port=14580)
    ```

    To run it in the background from a script:

    ```python
    from multiprocessing import Process
    from aerpawlib.v1.safety import SafetyCheckerServer

    def run_server():
        SafetyCheckerServer("config.yaml", 14580)  # blocks

    server_process = Process(target=run_server, daemon=True)
    server_process.start()
    ```

    Or use `screen`:

    ```bash
    screen -dmS safety_checker \
      python -m aerpawlib.v1.safety --port 14580 --vehicle_config config.yaml
    ```

    ### 3. Connect from your script

    ```python
    from aerpawlib.v1.safety import SafetyCheckerClient

    client = SafetyCheckerClient("127.0.0.1", 14580)
    ```

    ### Client methods

    #### `check_server_status() -> Tuple[bool, str]`

    ```python
    ok, msg = client.check_server_status()
    if ok:
        print("Server is running")
    ```

    #### `validate_takeoff_command(takeoffAlt, currentLat, currentLon) -> Tuple[bool, str]`

    ```python
    pos = drone.position
    ok, msg = client.validate_takeoff_command(10, pos.lat, pos.lon)
    if ok:
        await drone.takeoff(10)
    ```

    #### `validate_waypoint_command(curLoc, nextLoc) -> Tuple[bool, str]`

    ```python
    from aerpawlib.v1 import Coordinate

    current = Coordinate(35.7275, -78.6960, 10)
    target  = Coordinate(35.7280, -78.6955, 15)

    ok, msg = client.validate_waypoint_command(current, target)
    if ok:
        await drone.goto_coordinates(target)
    else:
        print(f"Target outside geofence: {msg}")
    ```

    #### `validate_change_speed_command(newSpeed) -> Tuple[bool, str]`

    ```python
    ok, msg = client.validate_change_speed_command(15.0)
    if ok:
        await drone.set_groundspeed(15.0)
    ```

    ### Full integration example

    ```python
    from aerpawlib.v1 import Drone, Coordinate, BasicRunner, entrypoint
    from aerpawlib.v1.safety import SafetyCheckerClient

    class SafeMission(BasicRunner):
        def __init__(self):
            self.checker = SafetyCheckerClient("127.0.0.1", 14580)

        @entrypoint
        async def run(self, drone: Drone):
            ok, msg = self.checker.check_server_status()
            if not ok:
                print(f"ERROR: Safety checker not running: {msg}")
                return

            pos = drone.position
            ok, msg = self.checker.validate_takeoff_command(10, pos.lat, pos.lon)
            if not ok:
                print(f"Cannot take off: {msg}")
                return

            await drone.takeoff(10)

            waypoints = [
                Coordinate(35.7275, -78.6960, 10),
                Coordinate(35.7280, -78.6955, 15),
            ]

            for wp in waypoints:
                ok, msg = self.checker.validate_waypoint_command(drone.position, wp)
                if ok:
                    await drone.goto_coordinates(wp)
                else:
                    print(f"Waypoint {wp} outside geofence: {msg}")
                    break

            await drone.land()
    ```

    ## v2 setup

    In v2, the safety client is wired into the vehicle at connection time. The `can_takeoff`, `can_goto`, and `can_land` methods on the vehicle automatically delegate to the safety client when one is configured.

    ### Via CLI (recommended)

    Pass `--safety-checker-port` when running the script. aerpawlib builds the client and passes it to `vehicle.connect(safety=...)` automatically.

    ```bash
    aerpawlib --api-version v2 --script my_mission.py \
      --vehicle drone --conn udpin://127.0.0.1:14550 \
      --safety-checker-port 14580
    ```

    | Environment | `--safety-checker-port` | Behavior |
    |-------------|------------------------|----------|
    | Non-AERPAW | Not provided | `NoOpSafetyChecker`: all validations pass, errors logged |
    | Non-AERPAW | Provided (e.g. `14580`) | Attempts to connect; on failure uses passthrough and logs |
    | AERPAW | Not provided | Defaults to port `14580`; connection failure is **fatal** |
    | AERPAW | Provided | Attempts to connect to given port; failure is **fatal** |

    ### Manually in Python

    ```python
    from aerpawlib.v2.safety import SafetyCheckerClient

    client = SafetyCheckerClient("127.0.0.1", 14580)
    drone = await Drone.connect("udpin://127.0.0.1:14550", safety=client)
    ```

    ### Command validation methods

    #### `can_takeoff(altitude) -> Tuple[bool, str]`

    Local checks: armable, GPS 3D fix, minimum battery. If `vehicle.safety` is set, also calls `safety.validate_takeoff`.

    ```python
    ok, msg = await drone.can_takeoff(10)
    if not ok:
        print(f"Cannot takeoff: {msg}")
        return
    await drone.takeoff(altitude=10)
    ```

    #### `can_goto(target) -> Tuple[bool, str]`

    Local checks: tolerance within valid range. If `vehicle.safety` is set, calls `safety.validate_waypoint`.

    ```python
    target = drone.position + VectorNED(20, 0, 0)
    ok, msg = await drone.can_goto(target)
    if not ok:
        print(f"Cannot go to target: {msg}")
        return
    await drone.goto_coordinates(target)
    ```

    #### `can_land() -> Tuple[bool, str]`

    If `vehicle.safety` is set, calls `safety.validate_landing`. Otherwise returns `(True, "")` unconditionally.

    ```python
    ok, msg = await drone.can_land()
    if ok:
        await drone.land()
    ```

    ### Full v2 integration example

    ```python
    from aerpawlib.v2 import BasicRunner, Drone, VectorNED, entrypoint
    from aerpawlib.v2.safety import PreflightChecks

    class ValidatedMission(BasicRunner):
        @entrypoint
        async def run(self, drone: Drone):
            # Preflight checks (GPS fix, battery)
            ok = await PreflightChecks.run_all(drone)
            if not ok:
                print("Preflight checks failed")
                return

            ok, msg = await drone.can_takeoff(10)
            if not ok:
                print(f"can_takeoff failed: {msg}")
                return

            await drone.takeoff(altitude=10)
            target = drone.position + VectorNED(20, 0, 0)

            ok, msg = await drone.can_goto(target)
            if not ok:
                print(f"can_goto failed: {msg}")
                await drone.land()
                return

            await drone.goto_coordinates(target)
            await drone.land()
    ```

## YAML config reference

```yaml
vehicle_type: copter       # 'copter' or 'rover'
max_speed: 15              # m/s
min_speed: 0.5             # m/s
max_alt: 120               # m (copter only)
min_alt: 5                 # m (copter only)
include_geofences:
  - flight_area.kml        # vehicle must stay inside these polygons
exclude_geofences:
  - no_fly_zone.kml        # vehicle must not enter these polygons
```

### Validation logic

The server validates requests in this order:

1. Waypoint position: must be inside at least one include geofence
2. No-go zones: must not be inside any exclude geofence
3. Path validation: path must not cross geofence boundaries
4. Path no-go: path must not enter any no-go zone
5. Altitude (copter): must be within `min_alt` and `max_alt`
6. Speed: must be within `min_speed` and `max_speed`

## KML geofence file format

Geofences are defined as KML polygons. Use QGroundControl or any GIS tool to export KML polygon files.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml/2.2">
  <Document>
    <Placemark>
      <name>Flight Area</name>
      <Polygon>
        <outerBoundaryIs>
          <LinearRing>
            <coordinates>
              -78.700,35.725,0
              -78.690,35.725,0
              -78.690,35.730,0
              -78.700,35.730,0
              -78.700,35.725,0
            </coordinates>
          </LinearRing>
        </outerBoundaryIs>
      </Polygon>
    </Placemark>
  </Document>
</kml>
```

<Note>
  KML coordinates use `longitude,latitude,altitude` order (note: longitude first).
</Note>

## Protocol details

The safety checker uses ZMQ REQ/REP with zlib-compressed JSON messages.

**Request:**
```json
{
  "request_function": "validate_waypoint_req",
  "params": [
    {"latitude": 35.7275, "longitude": -78.6960, "altitude": 10},
    {"latitude": 35.7280, "longitude": -78.6955, "altitude": 15}
  ]
}
```

**Response:**
```json
{
  "request_function": "validate_waypoint_req",
  "result": true,
  "message": ""
}
```

| Request type | Description |
|---|---|
| `server_status_req` | Check server is running |
| `validate_waypoint_req` | Validate a goto target |
| `validate_change_speed_req` | Validate a speed change |
| `validate_takeoff_req` | Validate takeoff altitude and position |
| `validate_landing_req` | Validate landing position |
