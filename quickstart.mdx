---
title: "Quickstart"
description: "Write and run your first aerpawlib mission in minutes."
---

This guide walks you from a fresh install to a drone that takes off, flies to a waypoint, and lands — using either the v1 or v2 API.

<Steps>
  <Step title="Install aerpawlib">
    If you haven't installed aerpawlib yet, follow the [installation guide](/installation). The short version:

    ```bash
    pip install -e .
    ```
  </Step>

  <Step title="Write your mission script">
    Create a file called `my_mission.py`. aerpawlib supports two runner styles: `BasicRunner` for linear scripts and `StateMachine` for phase-based missions.

    **BasicRunner** — run a sequence of commands top-to-bottom:

    <Tabs>
      <Tab title="v1 API">
        ```python my_mission.py
        from aerpawlib.v1 import Drone, Coordinate, BasicRunner, entrypoint

        class MyMission(BasicRunner):
            @entrypoint
            async def run(self, vehicle: Drone):
                await vehicle.takeoff(10)
                target = Coordinate(35.7275, -78.6960, 10)
                await vehicle.goto_coordinates(target)
                await vehicle.land()
        ```
      </Tab>
      <Tab title="v2 API">
        ```python my_mission.py
        from aerpawlib.v2 import BasicRunner, Drone, entrypoint

        class MyMission(BasicRunner):
            @entrypoint
            async def run(self, drone: Drone):
                await drone.takeoff(altitude=10)
                from aerpawlib.v2 import VectorNED
                target = drone.position + VectorNED(50, 0, 0)
                await drone.goto_coordinates(target)
                await drone.land()
        ```
      </Tab>
    </Tabs>

    **StateMachine** — define named states that transition into each other:

    <Tabs>
      <Tab title="v1 API">
        ```python my_mission.py
        from aerpawlib.v1 import Drone, StateMachine, state

        class MyMission(StateMachine):
            @state("takeoff", first=True)
            async def takeoff_state(self, vehicle: Drone):
                await vehicle.takeoff(10)
                return "land"

            @state("land")
            async def land_state(self, vehicle: Drone):
                await vehicle.land()
                return None  # end mission
        ```
      </Tab>
      <Tab title="v2 API">
        ```python my_mission.py
        from aerpawlib.v2 import Drone, StateMachine, state

        class MyMission(StateMachine):
            @state(name="takeoff", first=True)
            async def takeoff_state(self, drone: Drone):
                await drone.takeoff(altitude=10)
                return "land"

            @state(name="land")
            async def land_state(self, drone: Drone):
                await drone.land()
                return None  # end mission
        ```
      </Tab>
    </Tabs>

    <Note>
      Each script must define exactly one class that subclasses `BasicRunner` or `StateMachine`. For `BasicRunner`, exactly one method must be decorated with `@entrypoint`. For `StateMachine`, exactly one state must have `first=True`.
    </Note>
  </Step>

  <Step title="Run your script">
    Pass your script, connection string, and vehicle type to the `aerpawlib` CLI:

    <Tabs>
      <Tab title="v1 API">
        ```bash
        aerpawlib --api-version v1 \
          --script my_mission.py \
          --conn udpin://127.0.0.1:14550 \
          --vehicle drone
        ```
      </Tab>
      <Tab title="v2 API">
        ```bash
        aerpawlib --api-version v2 \
          --script my_mission.py \
          --conn udpin://127.0.0.1:14550 \
          --vehicle drone
        ```
      </Tab>
    </Tabs>

    | Flag | Required | Description |
    |------|----------|-------------|
    | `--script` | Yes | Path to your Python script |
    | `--conn` | Yes | MAVLink connection string (see [Installation](/installation)) |
    | `--vehicle` | Yes | `drone`, `rover`, or `none` |
    | `--api-version` | No | `v1` (default) or `v2` |
    | `--skip-init` | No | Skip pre-arm wait |
    | `--skip-rtl` | No | Do not RTL/land at script end |
    | `-v` / `-q` | No | Verbose (DEBUG) or quiet (WARNING) logging |

    <Tip>
      You can store all CLI flags in a JSON config file and pass it with `--config`:

      ```json mission.json
      {
        "script": "my_mission.py",
        "conn": "udpin://127.0.0.1:14550",
        "vehicle": "drone",
        "api_version": "v1"
      }
      ```

      ```bash
      aerpawlib --config mission.json
      ```

      CLI flags always override values in the config file.
    </Tip>
  </Step>
</Steps>

## A complete example

The `examples/` directory includes a square-pattern flight that demonstrates both APIs. It takes off, flies a 10 m × 10 m square using `VectorNED` offsets, and lands.

<Tabs>
  <Tab title="v1 API">
    ```python examples/v1/basic_example.py
    from aerpawlib.v1.runner import BasicRunner, entrypoint
    from aerpawlib.v1.util import VectorNED
    from aerpawlib.v1.vehicle import Drone


    class SquareFlight(BasicRunner):
        """Fly in a square pattern."""

        @entrypoint
        async def run(self, drone: Drone):
            takeoff_altitude = 10  # meters
            square_size = 10       # meters

            await drone.takeoff(takeoff_altitude)

            start_position = drone.position
            waypoints = [
                ("North", VectorNED(square_size, 0, 0)),
                ("East",  VectorNED(0, square_size, 0)),
                ("South", VectorNED(-square_size, 0, 0)),
                ("West",  VectorNED(0, -square_size, 0)),
            ]

            current_position = start_position
            for direction, offset in waypoints:
                target = current_position + offset
                await drone.goto_coordinates(target)
                current_position = target

            await drone.goto_coordinates(start_position)
            await drone.land()
    ```

    ```bash
    aerpawlib --api-version v1 \
      --script examples/v1/basic_example.py \
      --vehicle drone \
      --conn udp://127.0.0.1:14550
    ```
  </Tab>
  <Tab title="v2 API">
    ```python examples/v2/basic_example.py
    from aerpawlib.v2 import BasicRunner, Drone, VectorNED, entrypoint


    class SquareFlight(BasicRunner):
        """Fly in a square pattern."""

        @entrypoint
        async def run(self, drone: Drone):
            takeoff_altitude = 10
            square_size = 10

            await drone.takeoff(altitude=takeoff_altitude)

            start_position = drone.position
            waypoints = [
                ("North", VectorNED(square_size, 0, 0)),
                ("East",  VectorNED(0, square_size, 0)),
                ("South", VectorNED(-square_size, 0, 0)),
                ("West",  VectorNED(0, -square_size, 0)),
            ]

            current_position = start_position
            for direction, offset in waypoints:
                target = current_position + offset
                await drone.goto_coordinates(target)
                current_position = target

            await drone.goto_coordinates(start_position)
            await drone.land()
    ```

    ```bash
    aerpawlib --api-version v2 \
      --script examples/v2/basic_example.py \
      --vehicle drone \
      --conn udpin://127.0.0.1:14550
    ```
  </Tab>
</Tabs>
