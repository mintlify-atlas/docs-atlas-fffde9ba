---
title: "v1 Safety checker"
description: "SafetyCheckerServer and SafetyCheckerClient for geofence and constraint validation."
---

The safety checker system enforces geofence boundaries, altitude limits, and speed constraints before commands are sent to the vehicle. It runs as a separate process and communicates with your script over ZMQ using a REQ/REP pattern with zlib-compressed JSON messages.

<Info>
The v1 safety checker is functionally identical to the legacy version. All class and method names are unchanged.
</Info>

## Architecture

| Component | Role |
|-----------|------|
| `SafetyCheckerServer` | Loads config and geofences, validates commands, listens on a ZMQ REP socket. Constructor blocks — run in a dedicated process. |
| `SafetyCheckerClient` | Connects to the server from your script and calls validation methods before issuing vehicle commands. |

---

## Quick start

**1. Create a config file**

```yaml
# geofence_config.yaml
vehicle_type: copter
max_speed: 15
min_speed: 0.5
max_alt: 120
min_alt: 5
include_geofences:
  - flight_area.kml
exclude_geofences:
  - no_fly_zone.kml
```

**2. Start the server**

```bash
python -m aerpawlib.v1.safety --port 14580 --vehicle_config geofence_config.yaml
```

**3. Validate from your script**

```python
from aerpawlib.v1 import SafetyCheckerClient, Drone, BasicRunner, entrypoint

class SafeMission(BasicRunner):
    @entrypoint
    async def run(self, drone: Drone):
        checker = SafetyCheckerClient("127.0.0.1", 14580)

        ok, msg = checker.check_server_status()
        if not ok:
            print(f"Safety checker not running: {msg}")
            return

        ok, msg = checker.validate_takeoff_command(10, drone.position.lat, drone.position.lon)
        if not ok:
            print(f"Cannot take off: {msg}")
            return
        await drone.takeoff(10)
```

## SafetyCheckerServer

### Constructor

```python
SafetyCheckerServer(vehicle_config_filename: str, server_port: int = 14580)
```

Load the YAML config, parse geofence KML files, and start the blocking ZMQ REP loop.

<ParamField path="vehicle_config_filename" type="str" required>
  Path to the YAML configuration file.
</ParamField>

<ParamField path="server_port" type="int">
  Port to bind the ZMQ REP socket. Default: `14580`.
</ParamField>

<Warning>
The constructor **blocks indefinitely** — it runs the server loop until the process is terminated. Always start the server in a separate process or thread.
</Warning>

### Running the server

<Tabs>
  <Tab title="CLI (recommended)">
    ```bash
    python -m aerpawlib.v1.safety --port 14580 --vehicle_config geofence_config.yaml
    ```

    Run in the background with `screen`:

    ```bash
    screen -dmS safety python -m aerpawlib.v1.safety \
      --port 14580 --vehicle_config geofence_config.yaml
    ```
  </Tab>
  <Tab title="Python multiprocessing">
    ```python
    from multiprocessing import Process
    from aerpawlib.v1 import SafetyCheckerServer

    def run_server():
        SafetyCheckerServer("geofence_config.yaml", server_port=14580)

    server_process = Process(target=run_server, daemon=True)
    server_process.start()
    # Your mission code here...
    ```
  </Tab>
</Tabs>

### Validation logic

For each request the server checks, in order:

1. **Altitude** (copter only): next waypoint altitude must be within `[min_alt, max_alt]`.
2. **Geofence inclusion**: next waypoint must be inside at least one `include_geofences` polygon.
3. **Exclusion zones**: next waypoint must not be inside any `exclude_geofences` polygon.
4. **Path crossing**: the flight path from current to next position must not cross any geofence boundary.
5. **Path exclusion**: the flight path must not enter any no-go zone.
6. **Speed**: must be within `[min_speed, max_speed]`.

## SafetyCheckerClient

### Constructor

```python
SafetyCheckerClient(addr: str, port: int, timeout_s: float = 10.0)
```

<ParamField path="addr" type="str" required>
  IP address of the safety checker server, e.g. `"127.0.0.1"`.
</ParamField>

<ParamField path="port" type="int" required>
  Port the server is listening on.
</ParamField>

<ParamField path="timeout_s" type="float">
  Send/receive timeout in seconds. Prevents indefinite blocking if the server is down.
  Default: `10.0`.
</ParamField>

The client can also be used as a context manager to ensure the socket is closed:

```python
with SafetyCheckerClient("127.0.0.1", 14580) as checker:
    ok, msg = checker.check_server_status()
```

### `check_server_status`

```python
ok, msg = client.check_server_status() -> Tuple[bool, str]
```

Ping the server to confirm it is reachable and running.

<ResponseField name="ok" type="bool">``True`` if the server responded successfully.</ResponseField>
<ResponseField name="msg" type="str">Empty string on success, error description on failure.</ResponseField>

```python
ok, msg = checker.check_server_status()
if not ok:
    raise RuntimeError(f"Safety checker unavailable: {msg}")
```

### `validate_waypoint_command`

```python
ok, msg = client.validate_waypoint_command(
    current_location: Coordinate,
    next_location: Coordinate
) -> Tuple[bool, str]
```

Verify that the path from `current_location` to `next_location` respects all geofence and altitude constraints. Call this before every `goto_coordinates`.

<ParamField path="current_location" type="Coordinate" required>
  The vehicle's current position.
</ParamField>

<ParamField path="next_location" type="Coordinate" required>
  The intended target waypoint.
</ParamField>

```python
ok, msg = checker.validate_waypoint_command(drone.position, target)
if ok:
    await drone.goto_coordinates(target)
else:
    print(f"Waypoint rejected: {msg}")
```

### `validate_takeoff_command`

```python
ok, msg = client.validate_takeoff_command(
    takeoff_alt: float,
    current_lat: float,
    current_lon: float
) -> Tuple[bool, str]
```

Verify that `takeoff_alt` is within the configured altitude limits. Also records the takeoff location for subsequent landing validation.

<ParamField path="takeoff_alt" type="float" required>
  Intended takeoff altitude in meters.
</ParamField>

<ParamField path="current_lat" type="float" required>
  Current latitude (used to record the takeoff location).
</ParamField>

<ParamField path="current_lon" type="float" required>
  Current longitude.
</ParamField>

```python
pos = drone.position
ok, msg = checker.validate_takeoff_command(10, pos.lat, pos.lon)
if ok:
    await drone.takeoff(10)
```

### `validate_change_speed_command`

```python
ok, msg = client.validate_change_speed_command(new_speed: float) -> Tuple[bool, str]
```

Verify that `new_speed` is within the configured `[min_speed, max_speed]` range.

<ParamField path="new_speed" type="float" required>
  Requested speed in m/s.
</ParamField>

```python
ok, msg = checker.validate_change_speed_command(12.0)
if ok:
    await drone.set_groundspeed(12.0)
else:
    print(f"Speed rejected: {msg}")
```

### `validate_landing_command`

```python
ok, msg = client.validate_landing_command(
    current_lat: float,
    current_lon: float
) -> Tuple[bool, str]
```

Verify that the vehicle is within 5 meters of the recorded takeoff location before landing. Requires `validate_takeoff_command` to have been called first.

<ParamField path="current_lat" type="float" required>Current latitude.</ParamField>
<ParamField path="current_lon" type="float" required>Current longitude.</ParamField>

### `close`

```python
client.close()
```

Close the ZMQ socket and terminate the context. Not required if using the client as a context manager.

## YAML configuration format

```yaml
# geofence_config.yaml
vehicle_type: copter      # "copter" or "rover"
max_speed: 15             # Maximum allowed speed (m/s)
min_speed: 0.5            # Minimum allowed speed (m/s)
max_alt: 120              # Maximum altitude — copter only (meters)
min_alt: 5                # Minimum altitude — copter only (meters)
include_geofences:        # KML files defining allowed flight areas
  - flight_area.kml
exclude_geofences:        # KML files defining no-go zones
  - no_fly_zone.kml
```

| Parameter | Required for | Type | Description |
|-----------|-------------|------|-------------|
| `vehicle_type` | All | `str` | `"copter"` or `"rover"` |
| `max_speed` | All | `float` | Maximum speed in m/s |
| `min_speed` | All | `float` | Minimum speed in m/s |
| `max_alt` | Copter | `float` | Maximum altitude in meters |
| `min_alt` | Copter | `float` | Minimum altitude in meters |
| `include_geofences` | All | `list[str]` | Paths to allowed-area KML files |
| `exclude_geofences` | All | `list[str]` | Paths to no-go-zone KML files |

KML paths are resolved relative to the directory containing the YAML file.

## KML geofence format

Geofences are defined as KML files with a single `Polygon` element. Coordinates are in `longitude,latitude,altitude` order.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml/2.2">
  <Document>
    <Placemark>
      <name>Flight Area</name>
      <Polygon>
        <outerBoundaryIs>
          <LinearRing>
            <coordinates>
              -78.7,35.7,0
              -78.6,35.7,0
              -78.6,35.8,0
              -78.7,35.8,0
              -78.7,35.7,0
            </coordinates>
          </LinearRing>
        </outerBoundaryIs>
      </Polygon>
    </Placemark>
  </Document>
</kml>
```

<Note>
Coordinate order in KML is **longitude, latitude, altitude** — the reverse of the `Coordinate(lat, lon, alt)` constructor.
</Note>

## Full integration example

```python
from aerpawlib.v1 import Drone, Coordinate, BasicRunner, entrypoint, SafetyCheckerClient

class SafeMission(BasicRunner):
    def __init__(self):
        self.checker = SafetyCheckerClient("127.0.0.1", 14580)

    @entrypoint
    async def run(self, drone: Drone):
        # 1. Confirm server is available
        ok, msg = self.checker.check_server_status()
        if not ok:
            print(f"ERROR: safety checker not running: {msg}")
            return

        # 2. Validate takeoff
        pos = drone.position
        ok, msg = self.checker.validate_takeoff_command(10, pos.lat, pos.lon)
        if not ok:
            print(f"Cannot take off: {msg}")
            return
        await drone.takeoff(10)

        # 3. Fly to waypoints with validation
        waypoints = [
            Coordinate(35.7275, -78.6960, 10),
            Coordinate(35.7280, -78.6955, 15),
        ]
        for wp in waypoints:
            ok, msg = self.checker.validate_waypoint_command(drone.position, wp)
            if ok:
                await drone.goto_coordinates(wp)
            else:
                print(f"Waypoint {wp} rejected: {msg}")
                break

        await drone.land()
```
